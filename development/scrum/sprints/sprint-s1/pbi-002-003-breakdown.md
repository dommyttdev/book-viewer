# PBI-002 / PBI-003 ユーザーストーリー分解

## ストーリー

| PBI | GitHub Issue | ユーザ | 目的 | 価値 |
| --- | --- | --- | --- | --- |
| PBI-002 | #49 | 一般ユーザ | メール確認後に会員登録とPasskey登録を完了する。 | 登録メールアドレスの到達性を確認したうえで、パスワードなしで利用を開始できる。 |
| PBI-003 | #50 | 利用者 | Passkeyでログインし、安全なセッションを取得する。 | フィッシング耐性のある認証でログインし、ログイン状態をサーバ側で失効できる。 |

## 前提

- #89でPasskey / WebAuthn credential、登録 / 認証チャレンジ、セッション、復旧、初期super_admin登録方式を確定してから実装する。
- PBI-002とPBI-003はSprint S1の主導線として扱う。
- パスワードリセットは扱わない。Passkeyを使えない場合はPBI-004R / #90のアカウント復旧で扱う。
- 管理ユーザログインはPBI-003のセッション分離観点として考慮するが、管理ロール認可の本実装はSprint S2で扱う。

## PBI-002 受入条件ドラフト

- [ ] メールアドレス、表示名を入力して一般ユーザ登録を要求できる。
- [ ] メールアドレス形式が不正な場合は登録できない。
- [ ] 有効な一般ユーザのメールアドレスと重複する場合は登録できない。
- [ ] 登録直後の一般ユーザはメール未確認状態で保存される。
- [ ] メール確認用トークンは平文保存されず、ハッシュ値、有効期限、試行回数、再送回数を持つ。
- [ ] メール確認成功時にユーザをPasskey登録可能状態へ遷移し、使用したトークンを使用済みにする。
- [ ] 期限切れ、使用済み、失効済み、試行回数超過のメール確認トークンは拒否される。
- [ ] メール確認後にPasskey登録開始APIで登録チャレンジを取得できる。
- [ ] Passkey登録完了APIでcredential ID、公開鍵、sign count、transport、状態を保存できる。
- [ ] 登録チャレンジは平文保存されず、有効期限、使用済み、失効、試行回数を持つ。
- [ ] 開発環境では送信予定のメール確認リンクまたはトークンを安全な方法で確認できる。
- [ ] レスポンス、ログ、エラーに平文トークン、WebAuthnチャレンジ、credential秘密情報、セッションID、ハッシュ値を出力しない。

## PBI-003 受入条件ドラフト

- [ ] メール確認済みかつPasskey登録済みの一般ユーザはPasskeyログインを開始できる。
- [ ] メール未確認、Passkey未登録、停止、退会済みの一般ユーザはログイン完了できない。
- [ ] 認証開始APIはWebAuthn認証チャレンジを作成し、`challengeId`と認証オプションを返す。
- [ ] 認証チャレンジは平文保存されず、ハッシュ値、有効期限、試行回数、使用済み、失効状態を持つ。
- [ ] 認証完了APIはRP ID、origin、challenge、user presence、user verification、署名、必要に応じてsign countを検証する。
- [ ] 期限切れ、使用済み、失効済み、試行回数超過の認証チャレンジは拒否される。
- [ ] Passkey認証成功時に一般ユーザ用セッションを作成し、HTTP only Cookieを発行する。
- [ ] セッションCookieの平文値はDBへ保存せず、ハッシュ値で照合する。
- [ ] ログアウト時に現在のセッションを失効できる。
- [ ] 失効済み、期限切れ、主体が停止または退会済みのセッションでは認証できない。
- [ ] 一般ユーザ用セッションは管理ユーザ用セッションとして扱われない。
- [ ] レスポンス、ログ、エラーにWebAuthnチャレンジ、credential秘密情報、セッションID、ハッシュ値を出力しない。

## TDD開始観点

| PBI | Red候補 | 期待する失敗 |
| --- | --- | --- |
| PBI-002 | 登録要求が有効なメールと表示名を受け取り、未確認ユーザと確認トークンを作る。 | まだ永続化モデルやユースケースがなく失敗する。 |
| PBI-002 | 同じメールアドレスの有効ユーザがいる場合は登録できない。 | 重複判定が未実装で失敗する。 |
| PBI-002 | メール確認トークンは期限切れ、使用済み、試行回数超過を拒否する。 | トークン状態判定が未実装で失敗する。 |
| PBI-002 | Passkey登録チャレンジは期限切れ、使用済み、試行回数超過を拒否する。 | WebAuthn登録チャレンジ状態判定が未実装で失敗する。 |
| PBI-003 | メール未確認ユーザは認証チャレンジを作成できない。 | ユーザ状態判定が未実装で失敗する。 |
| PBI-003 | 正しいWebAuthn assertionでセッションが発行され、認証チャレンジが使用済みになる。 | credential検証とセッション発行が未実装で失敗する。 |
| PBI-003 | 失効済みセッションでは認証できない。 | セッション検証が未実装で失敗する。 |

## 実装対象外

- メールログインコード方式。
- 初期`super_admin`作成と管理ロール認可。
- Passkeyを使えない場合のアカウント復旧本実装。
- 端末ごとのセッション一覧。
- 本番メール送信基盤の詳細設定。
- TOTPや外部IDプロバイダ連携。
