## TDD観点

| 対象 | 最初に書くテスト | 種別 |
| --- | --- | --- |
| メール確認トークン | 平文トークンがDB保存値と一致せず、ハッシュ照合でのみ確認できる。 | 単体 |
| メール確認トークン | 期限切れ、使用済み、失効済み、試行回数超過を拒否する。 | 単体 / 結合 |
| ログインチャレンジ | `challengeId`でチャレンジを特定し、コードはハッシュ照合する。 | 単体 |
| ログインチャレンジ | メール未確認ユーザはチャレンジを作成できない。 | 結合 |
| セッション | Cookieの平文値はDBに保存されず、失効済みセッションでは認証できない。 | 単体 / 結合 |
| セッション分離 | 一般ユーザ用セッションを管理API主体として扱わない。 | 結合 |
| ログ安全性 | パスワード、トークン、コード、セッションID、ハッシュ値、pepperがログに出ない。 | 手動 / テスト |
