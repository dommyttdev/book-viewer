# テスト観点マトリクス

## 目的

このドキュメントは、PBIをTDDで実装する前に確認するテスト観点を整理する。

詳細なテスト戦略は [doc/06_testing/01_test_strategy.md](../../doc/06_testing/01_test_strategy.md) を参照する。

## PBI別テスト観点

| PBI | 単体テスト | 結合テスト | E2E / 手動確認 |
| --- | --- | --- | --- |
| PBI-001 プロジェクト基盤 | 設定読み込み、ヘルスチェック応答、モジュール境界、メール送信境界 | API / Worker起動、DBマイグレーション、DB接続 | ローカルミドルウェア起動確認、開発用メール確認 |
| PBI-002 ユーザ登録とメール確認 | 入力値、パスワード方針、トークン状態 | 登録API、メール確認API、重複、期限切れ、メール送信 | 登録からメール確認まで |
| PBI-003 ログイン2段階認証とセッション | challenge状態、code_hash、session_type、セッション失効 | 一般ログインAPI、管理ログインAPI、確認API、ログアウトAPI、メール送信 | 一般ログイン、管理ログイン、セッション分離確認 |
| PBI-004 パスワードリセット | トークン状態、既存セッション失効 | リセット要求API、確認API、存在推測防止、メール送信 | リセット後ログイン |
| PBI-005 初期管理ユーザと管理ロール認可 | 初期super_admin作成、ロール判定、最後のsuper_admin保護 | 管理APIアクセス制御、初期化再実行 | 権限別画面表示、初期ログイン確認 |
| PBI-006 書籍メタ情報モデル | 必須項目、公開状態、論理削除 | 永続化制約、関連保存 | 管理画面の表示確認 |
| PBI-007 RabbitMQ基盤 | メッセージ生成、冪等性キー、状態分離 | RabbitMQ配送、ack、再配送、DLQ | Worker停止時の滞留確認 |
| PBI-008 アップロード | ファイル形式判定、権限判定 | アップロードAPI、ジョブ作成 | 管理画面からアップロード |
| PBI-009 原本ファイル保存 | パス検証、保存名生成 | 一時ディレクトリ保存、失敗時ロールバック | 保存先確認 |
| PBI-010 zip縦切り変換 | アーカイブ内パス検証、品質値 | zip展開、WebP変換、ページ保存 | サンプルzip変換、可読性確認 |
| PBI-011 rar / 7zip展開 | コマンド組み立て、終了コード解釈 | 7-Zip実行、タイムアウト | Linux環境での実行確認 |
| PBI-012 サムネイルと変換状態 | 状態遷移、失敗理由、品質値 | サムネイル生成、状態API | 管理画面で状態確認 |
| PBI-013 メタ情報編集 | 入力値、タグ重複、シリーズ順序 | 書籍更新API、認可 | 管理画面から編集 |
| PBI-014 インデックスとOutbox | 検索ドキュメント生成、Outbox作成 | Elasticsearch登録、再構築、再試行 | 再インデックス手順 |
| PBI-015 検索 | クエリ条件生成、閲覧可能状態 | 検索API、日本語検索、ページング | 検索結果から詳細へ移動 |
| PBI-016 一覧と詳細 | ページング、サムネイル有無 | 一覧API、詳細API | 一覧から詳細へ移動 |
| PBI-017 ビューア | ページ境界、閲覧可否 | ページ画像取得API | ページ送り、スマートフォン表示 |
| PBI-018 読みかけ位置 | ページ範囲、ユーザ別保存 | 保存API、取得API | 再ログイン後の再開 |
| PBI-019 お気に入り | 重複防止、解除 | お気に入りAPI | 一覧から再閲覧 |
| PBI-020 本番相当Docker Compose統合 | 設定値、接続先 | API、Worker、Frontend、PostgreSQL、Elasticsearch、RabbitMQ接続、永続ボリューム | 本番相当構成の統合起動 |
| PBI-021 MVP E2E | 代表シナリオ分解 | 必要なAPI前提データ作成 | ログイン、アップロード、検索、閲覧、権限不足 |

## 共通異常系

すべての機能で、該当するものを確認する。

- 未認証
- 権限不足
- 存在しないリソース
- 入力不足
- 入力形式不正
- 重複
- 同時更新
- 外部依存失敗
- タイムアウト
- 秘密情報や不要な個人情報のログ出力

## 外部依存別の扱い

| 外部依存 | 基本方針 |
| --- | --- |
| PostgreSQL | 重要な永続化、制約、トランザクションは結合テストで確認する。 |
| Elasticsearch | クエリ生成は単体、実検索と再インデックスは結合テストで確認する。 |
| ファイルシステム | パス検証は単体、保存と削除は一時ディレクトリを使う結合テストで確認する。 |
| 7-Zip | コマンド組み立ては単体、実行は小さなサンプルで結合または手動確認する。 |
| 画像変換ライブラリ | 設定値と失敗時処理は単体、実画像変換は結合または手動確認する。 |
| ブラウザ | 主要導線はE2E、見た目と操作感は手動確認を併用する。 |

## スプリント内の確認記録

各PBI完了時に次を記録する。

| 項目 | 記録内容 |
| --- | --- |
| 先に書いたテスト | テスト名または観点 |
| 実行したテスト | コマンド、対象、結果 |
| 手動確認 | 環境、操作、結果 |
| 未確認事項 | 理由、リスク、次の対応 |
| ドキュメント更新 | 更新した `doc/` または `development/` のファイル |
