# 障害ログテンプレート

## 目的

このテンプレートは、本番運用中の障害、重大な運用対応、データ不整合、復旧判断を記録するために使用する。

## 記録対象

- サービス停止または大きな機能停止。
- PostgreSQL、Elasticsearch、専用キュー、書籍ファイル保存領域の障害。
- 原本ファイル、変換済みWebP、サムネイルの消失または不整合。
- 変換ジョブ失敗の大量発生。
- Elasticsearch全件再インデックスを必要とした不整合。
- ストレージ容量逼迫。
- セキュリティ上の懸念がある事象。
- 手順外の本番操作。

## 記録ルール

- 事実、判断、推測を分けて記録する。
- 秘密情報、トークン、パスワード、セッションID、内部物理パス、不要な個人情報は記録しない。
- 原因が未確定の場合は、未確定と明記する。
- バックアップなし運用により復元できない場合は、その判断と影響を明記する。
