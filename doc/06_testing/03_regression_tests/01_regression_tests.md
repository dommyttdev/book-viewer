# 回帰テスト一覧

## 目的

このドキュメントは、既存機能を変更したときに確認する回帰テスト対象を整理する。回帰テストはすべてを毎回手動で確認する一覧ではなく、変更範囲とリスクに応じて実行対象を選ぶための基準として扱う。

## 実行方針

- 不具合修正では、再発時に失敗する自動テストを先に追加する。
- 仕様変更では、影響を受ける受入テストと回帰テストを見直す。
- 重要フローはE2Eまたは手動確認で補い、細かな分岐は単体テストや結合テストで確認する。
- 実行したテスト、未実行の理由、既知リスクは作業記録またはプルリクエストへ残す。

## 更新方針

- 新しい受入テストを追加したら、必要に応じてこの一覧へ回帰観点を追加する。
- MVP実装が進んだら、常時実行する自動回帰テストとリリース前に実行する手動回帰テストを分ける。
- 不具合が発生した場合は、原因領域に回帰テストを追加し、再発防止テストの有無を記録する。
