# 画像変換機能の受入テスト

## 目的

アップロードされた原本ファイルからページ画像を安全に取り出し、WebP画像とサムネイルを生成し、変換ジョブ状態を追跡できることを確認する。

## 対象範囲

- 変換ワーカー
- RabbitMQ
- zip / rar / 7zip 展開
- 7-Zip for Linux コンソール版の外部プロセス実行
- WebP変換
- サムネイル生成
- 変換ジョブ状態更新
- 作業ディレクトリとパストラバーサル対策

## TDD観点

- ジョブ状態遷移はドメインまたはユースケースの単体テストで先に固定する。
- 7-Zip外部プロセスは、終了コード、タイムアウト、標準出力、標準エラーの扱いをインフラ境界でテストする。
- パストラバーサル、破損ファイル、空アーカイブは失敗するケースを先にテストする。

## 手動確認

- 実際の小さなzip、rar、7zipサンプルで変換完了を確認する。
- WebP画像とサムネイルの表示品質を確認する。
- 失敗ジョブのログと画面表示から、運用者が原因を追跡できることを確認する。

## 注意事項

実物の大容量原本ファイルや個人的な書籍データはGitへ含めない。負荷確認にはローカル専用データまたは生成データを使う。
