# 受入条件: お気に入り

## 対象ストーリー

- `FV-US-001`: 一般ユーザとして本をお気に入り登録したい
- `FV-US-002`: 一般ユーザとしてお気に入りを解除したい
- `FV-US-003`: 一般ユーザとしてお気に入り一覧を見たい
- `FV-US-004`: 一般ユーザとして重複登録を避けたい

## 受入条件

| ID | 条件 | 確認観点 |
| --- | --- | --- |
| FV-AC-001 | ログイン済み一般ユーザは閲覧可能な本をお気に入り登録できる。 | `POST /api/v1/books/{bookId}/favorite`で登録状態になること。 |
| FV-AC-002 | 未ログイン利用者はお気に入り登録、解除、一覧取得を実行できない。 | 401相当の認証エラーとなり、副作用がないこと。 |
| FV-AC-003 | 管理ユーザは一般ユーザ向けお気に入りを操作できない。 | 管理セッションから操作しても一般ユーザの`favorite`を作成しないこと。 |
| FV-AC-004 | 非表示、変換未完了、論理削除済みなど閲覧不可の本はお気に入り登録できない。 | 対象の存在や内部状態を過度に露出しないエラーになること。 |
| FV-AC-005 | 同一ユーザ、同一書籍の有効なお気に入りは一件までとする。 | 二重送信や再登録でも重複行が作成されないこと。 |
| FV-AC-006 | すでに登録済みの本を登録しようとした場合、登録済み状態として安全に扱う。 | APIは冪等に成功扱い、または409相当を返して画面が登録済みに整合できること。 |
| FV-AC-007 | ログイン済み一般ユーザは自分のお気に入りを解除できる。 | `DELETE /api/v1/books/{bookId}/favorite`で有効なお気に入りが解除されること。 |
| FV-AC-008 | 未登録のお気に入りを解除しようとした場合も、画面状態が未登録に整合する。 | APIは冪等に成功扱い、または404相当を返して画面が未登録に整合できること。 |
| FV-AC-009 | ログイン済み一般ユーザは自分のお気に入り一覧を取得できる。 | `GET /api/v1/me/favorites`で自分の有効なお気に入りのみ返ること。 |
| FV-AC-010 | お気に入り一覧には本一覧と同等の主要情報を表示できる。 | 表紙、タイトル、著者、タグ、シリーズ、読みかけ状態、お気に入り登録日時を表示できること。 |
| FV-AC-011 | 他ユーザのお気に入りは参照、登録、解除できない。 | ユーザIDを偽装できず、認証主体に紐づくデータだけを扱うこと。 |
| FV-AC-012 | お気に入り解除後、再登録できる。 | 論理削除または物理削除の方式にかかわらず、有効なお気に入りが一件になること。 |
| FV-AC-013 | お気に入り一覧は大量件数に備えてページングまたは追加読み込みできる。 | 一度に過大なデータを返さないこと。 |
| FV-AC-014 | APIレスポンスに物理パス、コンテナパス、`storage_key`を含めない。 | 画像はサムネイルURLなど公開API経由の参照だけを返すこと。 |

## 受入テストとの関係

お気に入りの主要フローは、回帰テスト一覧の閲覧、お気に入り対象に含める。詳細なE2Eケースは、画面実装時に本一覧、本詳細、お気に入り画面からの登録、解除、再閲覧として追加する。

## 未決事項

- 登録済みの再登録や未登録の解除を`200 OK`の冪等操作にするか、`409 Conflict` / `404 Not Found`を返して画面で整合させるかは、API実装時に最終決定する。
