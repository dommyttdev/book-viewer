# 受入条件: 本アップロード

## 対象ストーリー

- `BU-US-001`: 管理ユーザとして本をアップロードする
- `BU-US-002`: アップロード後に変換状態を確認する
- `BU-US-003`: 一般ユーザとして管理操作から保護される

## 受入条件

| ID | 条件 | 確認観点 |
| --- | --- | --- |
| BU-AC-001 | `book.upload`権限を持つ管理ユーザだけが本アップロードAPIを実行できる。 | 管理ユーザ、権限不足の管理ユーザ、一般ユーザ、未ログインで結果が分かれること。 |
| BU-AC-002 | 一般ユーザがアップロードAPIを呼び出した場合、本、原本ファイル、変換ジョブを作成しない。 | 403相当の認可エラーと副作用なし。 |
| BU-AC-003 | 未ログイン利用者がアップロードAPIを呼び出した場合、アップロード処理を開始しない。 | 401相当の認証エラー。 |
| BU-AC-004 | zip形式の原本アーカイブをアップロードできる。 | 原本ファイル保存、`book`登録、`book_file`登録、`conversion_job`作成、キュー投入。 |
| BU-AC-005 | rar形式の原本アーカイブをアップロードできる。 | 展開自体は変換ワーカーで行い、アップロード時点では対応形式として受け付ける。 |
| BU-AC-006 | 7zip形式の原本アーカイブをアップロードできる。 | 7-Zip for Linuxコンソール版で後続処理できる形式として扱う。 |
| BU-AC-007 | 許可されていない拡張子またはContent-Typeのファイルは拒否する。 | バリデーションエラーとなり、原本保存と変換ジョブ作成を行わない。 |
| BU-AC-008 | タイトルが未入力の場合は拒否する。 | バリデーションエラーとなり、原本保存と変換ジョブ作成を行わない。 |
| BU-AC-009 | アプリケーション独自の1冊あたりアップロードサイズ上限は設けない。 | 実行環境、Webサーバ、ストレージの制約は別途運用設定として扱う。 |
| BU-AC-010 | アップロードされた元ファイル名は物理ファイル名に直接使わない。 | 保存キーは内部IDベースにし、パストラバーサルや制御文字を無効化する。 |
| BU-AC-011 | アップロード成功時、APIは`201 Created`と書籍ID、変換ジョブID、初期状態を返す。 | `visibilityStatus`は`converting`、`conversionStatus`は`queued`を初期候補とする。 |
| BU-AC-012 | アップロード成功後、管理ユーザは変換ジョブ状態を確認できる。 | 画面または管理APIで変換待ち、処理中、完了、失敗を追跡できる。 |
| BU-AC-013 | 破損ファイルはアップロード受付後、変換処理で失敗状態として追跡できる。 | 失敗理由は安全な範囲で確認でき、秘密情報や内部パスを露出しない。 |
| BU-AC-014 | PostgreSQL登録またはジョブ作成に失敗した場合、書籍、原本ファイル、ジョブ状態の整合性が崩れない。 | 未参照ファイル検出またはロールバック可能な状態にする。 |
| BU-AC-015 | APIレスポンスに物理パス、コンテナパス、`storage_key`を含めない。 | クライアントにはリソースIDと状態だけを返す。 |
