# ユーザーストーリー: 本アップロード

## 目的

管理ユーザが自炊本アーカイブを登録し、一般ユーザがWeb上で読める状態へ変換処理を開始できるようにする。

## 背景

自炊本は、管理ユーザが原本アーカイブを登録し、変換ワーカーがWebP画像へ変換する。一般ユーザは書籍を保持せず、アップロードも行わないため、アップロードは管理領域の機能として扱う。

## 利用者

- 主利用者: 管理ユーザ
- 関係者: 一般ユーザ、運用者

## ストーリー

### BU-US-001: 管理ユーザとして本をアップロードする

管理ユーザとして、zip / rar / 7zip形式の自炊本アーカイブと基本メタ情報をアップロードしたい。なぜなら、一般ユーザがWeb上で本を読めるようにしたいから。

### BU-US-002: アップロード後に変換状態を確認する

管理ユーザとして、アップロード後に作成された変換ジョブの状態を確認したい。なぜなら、登録した本が閲覧可能になるまでの進捗や失敗理由を把握したいから。

### BU-US-003: 一般ユーザとして管理操作から保護される

一般ユーザとして、本アップロード操作を実行できないようにしてほしい。なぜなら、書籍管理は管理ユーザの責務であり、一般ユーザによる誤登録や不正なファイル登録を防ぎたいから。

## 対象範囲

- 管理ユーザによる原本アーカイブアップロード
- タイトルなどの基本メタ情報入力
- 原本ファイル保存
- PostgreSQLへの書籍、ファイル管理情報、変換ジョブ登録
- 専用キューへの変換ジョブ投入
- アップロード結果と変換ジョブ状態の表示
- 一般ユーザ、未ログイン利用者、権限不足の管理ユーザの拒否

## 対象外

- 変換ワーカー内部の展開、WebP変換、サムネイル生成の詳細
- 一般ユーザによるアップロード
- 外部書誌データベースからの自動補完
- 重複アップロードを禁止するかどうかの最終判断
