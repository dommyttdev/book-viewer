# ユーザーストーリー: 本メタ情報編集

## 目的

管理ユーザが登録済みの本のメタ情報を整え、一般ユーザが一覧、検索、詳細、閲覧導線で本を見つけやすくする。

## 背景

アップロード時点のメタ情報は最小限で登録されることがある。タイトル、著者、タグ、シリーズ、種別などは、検索と表示の品質に直接影響するため、管理ユーザが後から編集できる必要がある。

## 利用者

- 主利用者: 管理ユーザ
- 関係者: 一般ユーザ

## ストーリー

### BM-US-001: 管理ユーザとして基本メタ情報を編集する

管理ユーザとして、登録済みの本のタイトル、説明、著者、タグ、シリーズ、種別、シリーズ内順序を編集したい。なぜなら、一般ユーザが目的の本を見つけやすく、内容を判断しやすくしたいから。

### BM-US-002: 管理ユーザとして公開状態を制御する

管理ユーザとして、本の公開状態を管理したい。なぜなら、変換中、下書き、非表示、変換失敗などの本を一般ユーザの閲覧対象から適切に外したいから。

### BM-US-003: 管理ユーザとして検索更新の再試行余地を保つ

管理ユーザとして、検索対象のメタ情報を更新したときに、Elasticsearch更新が失敗してもPostgreSQLの更新内容を正として残したい。なぜなら、検索インデックスは派生データであり、後から再構築できるようにしたいから。

## 対象範囲

- タイトル、説明、著者、タグ、シリーズ、シリーズ内順序、種別の編集
- 公開状態の編集
- 入力値検証
- PostgreSQLへのメタ情報保存
- 検索対象項目変更時のElasticsearch更新または再試行可能な状態管理
- 一般ユーザと権限不足の管理ユーザの拒否

## 対象外

- 著者、タグ、シリーズ、種別のマスタ管理API
- 表記揺れ統合の詳細
- Elasticsearchのmapping、analyzer、再インデックス手順の詳細
- 表紙画像を手動で差し替える機能
