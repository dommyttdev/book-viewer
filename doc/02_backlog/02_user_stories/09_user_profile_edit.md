# ユーザーストーリー: 会員情報編集

## 目的

一般ユーザが自分の会員情報を確認し、表示名などのプロフィール情報を安全に更新できるようにする。

## 背景

一般ユーザの会員情報編集は、自分自身の情報だけを対象とする。バックエンドAPIは、フロントエンドの表示制御に依存せず、認証済みの一般ユーザが自分の`/me`相当の情報だけを取得、更新できることを検証する。

初版では表示名の更新を主対象とする。メールアドレス変更を提供する場合は、変更先メールアドレスの確認、変更前後への通知、ログインID変更の扱いを別途定義する。

## 利用者

- 主利用者: 一般ユーザ

## ストーリー

### AM-US-010: 一般ユーザとして会員情報を確認したい

一般ユーザとして、自分のメールアドレス、表示名、状態、メール確認状態を確認したい。なぜなら、登録内容が正しいか把握したいから。

### AM-US-011: 一般ユーザとして表示名を変更したい

一般ユーザとして、自分の表示名を変更したい。なぜなら、アプリ内で使う名前を現在の好みに合わせたいから。

### AM-US-012: 一般ユーザとして自分以外の情報を操作できないようにしたい

一般ユーザとして、他の利用者の会員情報を参照または更新できないようにしてほしい。なぜなら、個人情報とアカウントの安全性を守りたいから。

### AM-US-013: 一般ユーザとして変更内容を検証してほしい

一般ユーザとして、表示名などの入力値を更新前に検証してほしい。なぜなら、不正な値や想定外の長さでプロフィールが壊れないようにしたいから。

## 対象範囲

- 自分の会員情報取得
- 表示名の更新
- 入力値検証
- 自分以外の会員情報へのアクセス防止
- 停止、退会済みユーザの更新制御

## 対象外

- メールアドレス変更
- パスワード変更
- プロフィール画像
- 公開プロフィール
- 管理ユーザによる一般ユーザ編集

## 受入条件

- 認証済み一般ユーザは、自分の会員情報を取得できる。
- 会員情報取得レスポンスにパスワードハッシュ、認証トークン、セッションIDを含めない。
- 認証済み一般ユーザは、自分の表示名を更新できる。
- 表示名は長さ、空文字、制御文字などを検証できる。
- 一般ユーザは他ユーザの会員情報を参照または更新できない。
- 停止または退会済みユーザは会員情報更新を完了できない。
- メールアドレス変更は初版の対象外として扱い、提供する場合は追加設計を行う。
