# ユーザーストーリー: 会員登録

## 目的

未ログイン利用者が一般ユーザとして会員登録し、メール確認を完了したうえで本の閲覧、検索、お気に入りなどを利用できるようにする。

## 背景

このアプリでは一般ユーザは書籍を保持せず、管理ユーザが登録した共通の書籍カタログを閲覧する。会員登録では、本人が管理できるメールアドレスであることを確認し、パスワードを安全に扱い、登録済みメールアドレスの有無を第三者に推測されにくい応答にする。

メール確認が完了するまで、通常の一般ユーザ向け機能は利用できない。確認メールの再送、トークン有効期限、登録途中アカウントの扱いは、実装時にレート制限と運用方針を合わせて具体化する。

## 利用者

- 主利用者: 未ログイン利用者
- 関係者: 一般ユーザ

## ストーリー

### AM-US-001: 未ログイン利用者として会員登録したい

未ログイン利用者として、メールアドレス、パスワード、表示名を入力して会員登録したい。なぜなら、自分の閲覧履歴やお気に入りを利用できる一般ユーザアカウントを作りたいから。

### AM-US-002: 未ログイン利用者としてメールアドレスを確認したい

未ログイン利用者として、登録後に届く確認メールからメール確認を完了したい。なぜなら、本人が利用できるメールアドレスで登録していることを証明し、通常利用を開始したいから。

### AM-US-003: 未ログイン利用者として確認メールを再送したい

未ログイン利用者として、確認メールが届かない場合に再送したい。なぜなら、メール配送遅延や誤削除があっても登録を完了できるようにしたいから。

### AM-US-004: 利用者として安全な登録エラーを受け取りたい

利用者として、入力不備や登録済みメールアドレスの扱いについて安全なエラーを受け取りたい。なぜなら、修正すべき内容を理解しつつ、他人のアカウント存在を推測できない状態にしたいから。

## 対象範囲

- 一般ユーザの会員登録
- メールアドレス形式、パスワード強度、表示名の入力検証
- 会員登録時のメール認証
- メール確認トークンの有効期限、使い捨て、再送制限
- メール確認完了までの利用制限
- 登録済みメールアドレスの存在推測を避ける応答

## 対象外

- 管理ユーザの登録
- 外部IDプロバイダ連携
- 一般ユーザによる書籍アップロード
- メールアドレス変更フロー

## 受入条件

- 未ログイン利用者は、メールアドレス、パスワード、表示名で一般ユーザ登録を要求できる。
- メールアドレス形式とパスワード強度を検証できる。
- 登録成功後はメール確認待ち状態になり、通常利用は許可されない。
- メール確認トークンは有効期限と使い捨て性を持つ。
- 確認メールの再送には過剰送信を防ぐ制限を設けられる。
- 登録済みメールアドレスに関する応答は、アカウント存在を推測されにくい。
- パスワード、確認トークン、セッションIDはレスポンスやログへ出力しない。
