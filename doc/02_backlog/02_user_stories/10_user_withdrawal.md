# ユーザーストーリー: 退会

## 目的

一般ユーザが自分のアカウントを退会状態へ変更し、退会後にログインや個人機能を利用できないようにする。

## 背景

一般ユーザは書籍を保持しないため、退会時にアップロード済み書籍、原本ファイル、変換済みWebP、サムネイルを削除する必要はない。退会時の主な対象は、一般ユーザ情報、お気に入り、閲覧履歴、セッションである。

退会は取り消しが難しい操作として扱い、本人確認を行ったうえで実行する。退会後のメールアドレス再利用可否は、メール一意制約、監査要件、個人情報削除方針を踏まえて実装時に決定する。

## 利用者

- 主利用者: 一般ユーザ

## ストーリー

### AM-US-014: 一般ユーザとして退会したい

一般ユーザとして、自分のアカウントを退会したい。なぜなら、アプリの利用を終了し、個人機能を停止したいから。

### AM-US-015: 一般ユーザとして退会前に本人確認したい

一般ユーザとして、退会実行前にパスワードなどで本人確認してほしい。なぜなら、誤操作や第三者による不正な退会を防ぎたいから。

### AM-US-016: 一般ユーザとして退会後にログインできないようにしたい

一般ユーザとして、退会後は同じアカウントでログインできないようにしてほしい。なぜなら、利用終了後のアカウント状態を明確にしたいから。

### AM-US-017: 一般ユーザとして退会時の個人機能データを整理してほしい

一般ユーザとして、退会時にお気に入りや閲覧履歴を無効化してほしい。なぜなら、退会後に個人機能のデータが通常利用に残らないようにしたいから。

## 対象範囲

- 一般ユーザ自身の退会
- 退会前の本人確認
- ユーザ状態の`withdrawn`への遷移
- 退会時のセッション無効化
- お気に入りと閲覧履歴の無効化
- 退会後のログイン拒否
- 退会時のアップロード済み書籍削除が対象外であることの明確化

## 対象外

- 管理ユーザの削除または停止
- 一般ユーザが保持する書籍の削除
- 退会済みアカウントの復元
- 退会後メールアドレス再利用可否の最終決定

## 受入条件

- 認証済み一般ユーザは、自分のアカウントだけを退会できる。
- 退会実行前にパスワードなどで本人確認できる。
- 退会成功後、ユーザ状態は`withdrawn`へ遷移し、既存セッションは無効化される。
- 退会済みユーザはログインできない。
- 退会時にお気に入りと閲覧履歴を無効化できる。
- 一般ユーザは書籍を保持しないため、退会時のアップロード済み書籍削除は対象外である。
- 書籍メタ情報、原本ファイル、変換済みWebP、サムネイルは一般ユーザ退会で影響を受けない。
- 監査やセキュリティイベントに必要な情報は、必要最小限の範囲で保持できる。
