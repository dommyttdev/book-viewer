# ユーザーストーリー: 管理ユーザログイン

## 目的

管理ユーザが一般ユーザ領域とは分離された管理領域へ安全にログインし、利用終了時に管理セッションを無効化できるようにする。

## 背景

管理ユーザは書籍アップロード、管理ユーザ管理、ロール設定、一般ユーザ停止など影響の大きい操作を行う。そのため、一般ユーザと同じメールアドレス、パスワード、メール2段階認証を前提にしつつ、管理ログイン入口、管理セッション、権限確認は一般ユーザ向け機能と分けて扱う。

管理ユーザの停止、削除、ロール変更があった場合は、既存の管理セッションを失効できる設計にする。認証失敗や2段階認証失敗では、アカウント存在や管理者であることを推測されにくい応答にする。

## 利用者

- 主利用者: 管理ユーザ
- 関係者: 運用者

## ストーリー

### AD-US-001: 管理ユーザとして管理ログインしたい

管理ユーザとして、管理ログイン画面からメールアドレスとパスワードを入力してログインを開始したい。なぜなら、管理領域で自分に許可された管理操作を実行したいから。

### AD-US-002: 管理ユーザとしてメール2段階認証を完了したい

管理ユーザとして、メールで届くワンタイムコードまたは確認リンクで2段階認証を完了したい。なぜなら、管理操作をパスワードだけに依存させず安全に扱いたいから。

### AD-US-003: 管理ユーザとして管理ログアウトしたい

管理ユーザとして、管理領域の利用終了時に管理セッションを無効化したい。なぜなら、共有端末や離席時に管理権限を悪用されないようにしたいから。

### AD-US-004: 管理ユーザとして権限に応じた管理画面へ入りたい

管理ユーザとして、ログイン後に自分のロールで許可された画面と操作だけを利用したい。なぜなら、必要な管理作業を行いつつ、権限外の操作を誤って実行しないようにしたいから。

### AD-US-005: 管理ユーザとして安全な認証エラーを受け取りたい

管理ユーザとして、認証失敗や権限不足時に安全で分かりやすいエラーを受け取りたい。なぜなら、修正すべき操作を理解しつつ、管理アカウントの存在を第三者に推測させたくないから。

## 対象範囲

- 管理ユーザ用ログイン開始
- 管理ユーザ用メール2段階認証
- 管理ユーザ用セッションの発行と無効化
- 管理ログアウト
- 停止、削除、ロール変更済み管理ユーザのログイン制御
- ログイン失敗、2段階認証失敗、再送要求の制限
- ロールに応じた管理画面表示と管理API権限確認

## 対象外

- 一般ユーザログイン
- 外部IDプロバイダ連携
- 管理ユーザの自己登録
- 端末ごとの管理セッション一覧
- 管理ユーザ招待と初回パスワード設定の詳細

## 受入条件

- 有効な管理ユーザだけが管理ログインを開始できる。
- パスワード認証成功後、メールによる2段階認証を要求できる。
- 2段階認証のコードまたは確認リンクは有効期限と使い捨て性を持つ。
- 管理ログイン成功時は、一般ユーザ用セッションと分離された管理ユーザ用セッションを発行できる。
- 管理ログアウト時は現在の管理セッションを無効化できる。
- 停止、削除、メール未確認、権限なしの管理ユーザは管理ログインを完了できない。
- ロール変更や停止後は、必要に応じて既存の管理セッションを失効できる。
- 認証失敗理由は、管理アカウント存在や内部状態を過度に漏らさない。
- 管理APIはフロントエンド表示制御に依存せず、サーバ側で管理ユーザ認証と権限を確認できる。
