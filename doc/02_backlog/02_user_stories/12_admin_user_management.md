# ユーザーストーリー: 管理ユーザ管理

## 目的

`super_admin` が管理ユーザを登録、編集、停止または削除し、管理権限を持つ利用者を安全に維持できるようにする。

## 背景

管理ユーザは一般ユーザよりも強い操作権限を持つため、作成、変更、停止は影響範囲が大きい。初版では、管理ユーザ管理は `super_admin` のみが実行できるものとし、`admin`, `operator`, `viewer` は管理ユーザの登録、編集、削除を行えない。

管理ユーザ削除は監査と過去操作主体の参照を維持するため、物理削除ではなく論理削除または停止を基本とする。最後の `super_admin` を停止または権限剥奪する操作は、運用不能を避けるため拒否する。

## 利用者

- 主利用者: `super_admin`
- 関係者: 管理ユーザ、運用者

## ストーリー

### AD-US-006: super_admin として管理ユーザを登録したい

`super_admin` として、新しい管理ユーザを登録したい。なぜなら、書籍管理、変換運用、参照業務を担当する管理者を安全に追加したいから。

### AD-US-007: super_admin として管理ユーザ一覧を確認したい

`super_admin` として、管理ユーザのメールアドレス、表示名、状態、ロール、最終ログイン日時を一覧したい。なぜなら、管理権限を持つ利用者を把握し、不要なアカウントを発見したいから。

### AD-US-008: super_admin として管理ユーザを編集したい

`super_admin` として、管理ユーザの表示名、状態、ロールを編集したい。なぜなら、担当変更や権限変更を管理画面から反映したいから。

### AD-US-009: super_admin として管理ユーザを停止または削除したい

`super_admin` として、不要または危険な管理ユーザを停止または削除したい。なぜなら、退職、担当終了、漏えい疑いがある場合に管理領域へのアクセスを止めたいから。

### AD-US-010: super_admin として危険な管理ユーザ操作を防ぎたい

`super_admin` として、最後の `super_admin` 停止や自分自身の権限喪失など運用不能につながる操作を拒否してほしい。なぜなら、誤操作によって管理機能を復旧できない状態にしたくないから。

## 対象範囲

- 管理ユーザ一覧、検索、絞り込み
- 管理ユーザ登録
- 管理ユーザ表示名、状態、ロールの編集
- 管理ユーザ停止、復帰、論理削除
- 最後の `super_admin` 保護
- 管理ユーザの重複メール検証
- 管理ユーザ管理操作のサーバ側権限確認
- 必要な範囲の監査ログ記録候補

## 対象外

- 一般ユーザの会員登録、編集、退会
- 一般ユーザ停止の詳細運用
- 管理ユーザ自身によるセルフサービス登録
- 初回パスワード設定、招待メール本文、招待期限の詳細
- 外部IDプロバイダ連携
- 詳細な監査ログ画面

## 受入条件

- `super_admin` は管理ユーザを一覧、登録、編集、停止または削除できる。
- `admin`, `operator`, `viewer`, 一般ユーザは管理ユーザを登録、編集、削除できない。
- 管理ユーザ登録時はメールアドレス形式、表示名、ロールを検証できる。
- 管理ユーザ登録時に平文パスワードを管理者へ通知しない。
- 管理ユーザ編集時は最後の `super_admin` を失わせる変更を拒否できる。
- 管理ユーザ停止または削除時は既存の管理セッションを失効できる。
- 管理ユーザ削除は論理削除または停止を基本とし、過去操作主体の参照を壊さない。
- 管理ユーザ管理操作はバックエンドAPI側で権限確認できる。
- パスワード、セッションID、2段階認証コード、招待トークンはレスポンスやログに出力しない。
