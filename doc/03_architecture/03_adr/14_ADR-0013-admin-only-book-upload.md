# ADR-0013: 一般ユーザは書籍を保持せず管理ユーザのみアップロード可能にする

## Status

Accepted

## Context

本システムは、管理ユーザがアップロードした自炊本をWeb閲覧向けに変換し、一般ユーザがブラウザから検索、閲覧、お気に入り管理できる状態にすることを目的とする。

一般ユーザ向け機能は、本一覧、検索、ビューア、お気に入り、閲覧履歴、会員情報管理を中心とする。管理ユーザ向け機能は、書籍アップロード、メタ情報管理、変換ジョブ確認、再実行、一般ユーザ管理、ロール管理を中心とする。

書籍アップロード、アーカイブ展開、WebP変換、サムネイル生成は、容量、著作物管理、変換失敗、セキュリティ、運用対応への影響が大きい。アップロードファイル、アーカイブ内パス、展開後総量、画像ピクセル数、ジョブ一時領域には安全上限が必要であり、失敗時の調査や再実行も管理操作として扱う必要がある。

一般ユーザごとに書籍所有を許可すると、所有者境界、共有可否、容量上限、退会時の書籍削除、管理者によるユーザ所有書籍の閲覧や削除、検索結果の公開範囲が複雑になる。初期方針では、登録済みの書籍を一般ユーザが読むモデルを優先し、一般ユーザ所有の書籍管理は扱わない。

## Decision

一般ユーザは書籍を保持しない。書籍アップロードは管理ユーザのみが実行できる。

`book` と `book_file` の作成主体は管理ユーザとし、必要に応じてアップロードした管理ユーザを記録する。一般ユーザには書籍所有権を持たせず、`book` から一般ユーザへの所有者関連は作成しない。

一般ユーザは、有効かつ閲覧可能な書籍を一覧、検索、閲覧できる。お気に入りと閲覧履歴は一般ユーザ自身に属するデータとして管理するが、対象書籍そのものは一般ユーザの所有物ではない。

一般ユーザ同士の書籍共有機能は提供しない。一般ユーザが書籍を保持しないため、共有元、共有先、共有権限、共有解除、共有範囲検索は初期設計の対象外とする。

管理者による一般ユーザ所有書籍の閲覧、削除、退会時のアップロード済み書籍削除は対象外とする。管理ユーザの `book.view` は、登録済み管理対象書籍の内容確認、変換品質確認、ページ順確認のための権限であり、一般ユーザ所有書籍を閲覧する権限ではない。

退会時は、一般ユーザ情報、セッション、お気に入り、閲覧履歴を対象に扱う。一般ユーザは書籍を保持しないため、退会に伴う書籍メタ情報、原本ファイル、変換済みWebP、サムネイルの削除は行わない。

## Consequences

- 書籍アップロード、変換、削除、再実行を管理ユーザの責務として集約できる。
- 書籍所有者ごとの権限、共有、容量上限、検索公開範囲を初期設計から外せる。
- データモデルでは、書籍と一般ユーザの所有関係を持たせず、一般ユーザに紐づくのはお気に入りと閲覧履歴に限定できる。
- 権限設計では、一般ユーザは閲覧中心、管理ユーザは書籍管理中心として責務を明確にできる。
- 一般ユーザ退会時に、アップロード済み書籍や生成ファイルを削除する必要がない。
- 一般ユーザが自分だけの書籍をアップロードして読むユースケースは初期対象外になる。
- 管理ユーザがアップロードする書籍は、公開状態や閲覧可能状態を通じて一般ユーザへの表示可否を制御する必要がある。

## Alternatives

- 一般ユーザごとに書籍所有を許可する。
  - 個人ライブラリとして自然だが、容量上限、所有者権限、検索範囲、退会時削除、変換ジョブ負荷、管理者対応が複雑になるため初期方針では採用しない。
- 一般ユーザ同士の書籍共有を提供する。
  - 共有ライブラリとしての利便性はあるが、共有権限、公開範囲、共有解除、通報や削除、検索結果の可視性が複雑になるため採用しない。
- 管理者が一般ユーザの代わりに所有書籍をアップロードする。
  - アップロード操作を管理者に限定しつつ個人所有を表現できるが、所有者関連、退会時削除、管理者による所有書籍閲覧の扱いが残る。初期方針では所有者を一般ユーザに持たせない。
- 全書籍を管理者だけが閲覧し、一般ユーザ閲覧を提供しない。
  - 管理用アーカイブとしては単純だが、本システムの目的である一般ユーザの検索、閲覧、お気に入り管理に反するため採用しない。
- 管理ユーザにも書籍所有を持たせず、アップロード主体を記録しない。
  - モデルは単純になるが、問い合わせ、監査、変換失敗時の確認、運用上の責任追跡が弱くなるため、アップロードした管理ユーザは記録できる方針とする。
