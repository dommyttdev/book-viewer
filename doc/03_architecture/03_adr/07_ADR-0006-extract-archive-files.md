# ADR-0006: 7-Zip for Linuxコンソール版でアーカイブを展開する

## Status

Accepted

## Context

本システムでは、管理ユーザがアップロードしたzip / rar / 7zip形式の自炊本アーカイブを展開し、ページ画像を取り出す。

zipはJava標準ライブラリでも扱えるが、rarや7zipへの対応を安定させるには別の手段が必要になる。Java標準ライブラリやApache Commons Compressのrar対応に依存すると、対応形式や暗号化、破損ファイル、実運用での互換性に制約が出る可能性がある。

アーカイブ展開はパストラバーサル、巨大ファイル、破損ファイル、暗号化アーカイブなどのリスクを伴うため、変換ワーカーの責務として隔離し、外部プロセス制御と検証を明示する必要がある。

## Decision

zip / rar / 7zip形式のアーカイブ展開には、7-Zip for Linuxコンソール版を使用する。

7-Zipは変換ワーカーコンテナ内で利用し、Spring Boot変換ワーカーから外部プロセスとして呼び出す。

展開はジョブごとの専用作業ディレクトリで行う。実行ファイルパス、引数、作業ディレクトリ、タイムアウト、終了コード、標準出力、標準エラーを制御し、結果を変換ジョブ状態へ反映する。

アーカイブ内パスと展開先パスは検証し、パストラバーサルを防ぐ。破損ファイル、暗号化アーカイブ、非対応形式の扱いは画像変換設計で詳細化する。

## Consequences

- zip / rar / 7zipへの対応を、Javaライブラリの対応範囲に依存せず統一できる。
- 変換ワーカーに展開処理を閉じ込めることで、HTTP APIの応答性と責務分離を保ちやすい。
- 外部プロセスの終了コード、標準エラー、タイムアウトをジョブ診断情報として扱える。
- 7-Zip実行ファイルの導入、バージョン管理、コンテナ内配置、実行権限を運用で管理する必要がある。
- 外部プロセス呼び出しの入力、引数、作業ディレクトリ、ログ出力にはセキュリティ上の注意が必要になる。
- 展開後のクリーンアップ、巨大ファイル、一時ディスク使用量、暗号化アーカイブの扱いは後続設計で明確化する必要がある。

## Alternatives

- Java標準ライブラリのみで展開する。
  - zipには対応できるが、rar / 7zip対応が不足するため採用しない。
- Apache Commons CompressなどのJavaライブラリで展開する。
  - Java内で完結できるが、rar対応や実運用上の互換性を重視し、今回は採用しない。
- アップロード可能形式をzipのみに限定する。
  - 実装は単純になるが、自炊本アーカイブとしてrar / 7zipも扱いたいため採用しない。
- 7-ZipをAPIプロセスから直接呼び出す。
  - 実装は近道になるが、重い処理をAPIから分離しにくくなるため、変換ワーカーから呼び出す。
