# ADR-0011: フロントエンドにNext.jsを採用する

## Status

Accepted

## Context

本システムは、一般ユーザ向けの本一覧、検索、ビューア、お気に入り管理と、管理ユーザ向けの書籍アップロード、メタ情報編集、変換ジョブ確認、ユーザ管理を同一のWebフロントエンドで提供する。

一般ユーザ向け画面では、サムネイルや変換済みWebP画像を使った閲覧体験、検索条件やページング、閲覧位置の保存、お気に入り状態などを扱う。管理ユーザ向け画面では、アップロード進行、非同期変換ジョブ状態、失敗時の再実行導線、メタ情報編集、管理権限によるメニュー制御を扱う。

認証状態は、HTTP onlyかつSecure属性付きCookieによるサーバ側セッションを第一候補とする。フロントエンドはCookieを利用してバックエンドAPIを呼び出すが、認証、認可、入力値、アップロードファイル、アーカイブ内容の最終検証はSpring BootバックエンドAPIで行う必要がある。

初期運用は単一Linuxホスト上のDocker Compose構成であり、フロントエンド、API、変換ワーカー、PostgreSQL、Elasticsearch、RabbitMQを同一ホスト上で運用する。将来的にAPIやミドルウェアを分離しても、フロントエンドはAPI境界を通じて疎結合に保てる構成が望ましい。

## Decision

フロントエンド技術としてNext.jsを採用する。

Next.jsフロントエンドWebアプリケーションは、一般ユーザ向け画面と管理ユーザ向け画面を同一アプリケーション内で提供する。画面、レイアウト、ルーティング、入力補助、API呼び出し、画面状態管理、変換待ちやエラー状態の表示を担当する。

バックエンドAPIとの通信はHTTP APIを通じて行う。認証CookieはブラウザとAPIの間で利用し、フロントエンドはログイン状態に応じた画面遷移や表示制御を行う。ただし、権限判断の正本はバックエンドAPIとPostgreSQLに置き、フロントエンドの表示制御をセキュリティ境界とはみなさない。

画像ビューアでは、原本ファイルではなく変換済みWebP画像とサムネイルを表示する。ページ一覧、ページ画像取得、閲覧位置保存、お気に入り更新はAPIを介して行い、ページングや分割取得を前提にする。

一般ユーザログインと管理ユーザログインは、画面入口、遷移先、セッション種別を分けて扱う。実装上のルーティングやディレクトリ構成は後続のフロントエンド実装で具体化するが、同一Next.jsアプリケーション内で利用者種別ごとの画面責務を分離する。

## Consequences

- 一般ユーザ向け画面と管理ユーザ向け画面を同じ技術スタック、同じビルドとデプロイ単位で提供できる。
- API呼び出し、画面状態管理、エラー表示、変換待ち表示をフロントエンド側で一貫して扱いやすい。
- サーバ側セッションCookieを利用するため、CSRF対策、SameSite属性、CORS設定、HTTPS前提をAPI契約とセキュリティ設計で継続して確認する必要がある。
- 画像ビューアはAPIと変換済みWebPに依存するため、未変換、変換失敗、ページ欠損、閲覧不可状態を明示的に扱う必要がある。
- 管理画面を同一アプリケーションで提供するため、ルーティング、メニュー表示、API呼び出しで一般ユーザ領域と管理ユーザ領域を混同しない設計が必要になる。
- フロントエンドの入力検証は利便性のための補助であり、セキュリティとデータ整合性はバックエンドAPIで担保する。

## Alternatives

- React SPAを採用する。
  - Reactエコシステムを活用でき、単純なクライアントアプリとして構築しやすい。一方で、ルーティング、ビルド、データ取得、将来的なレンダリング方針を個別に選定する範囲が増えるため、初期方針ではNext.jsを採用する。
- Vue / Nuxtを採用する。
  - NuxtもWebアプリケーション構築に有力だが、現時点の計画ではNext.jsを前提にしており、React系のコンポーネント資産とチーム習熟性を優先する。
- Spring MVC / Thymeleafでサーバレンダリングする。
  - Spring Bootとの統合は単純になるが、ビューア操作、検索結果、変換状態、管理画面の画面状態管理が肥大化しやすい。フロントエンドとAPIの分離余地も小さくなるため採用しない。
- 一般ユーザ向け画面と管理ユーザ向け画面を別フロントエンドに分ける。
  - 権限境界を見た目上分けやすいが、初期運用ではビルド、デプロイ、共通UI、認証連携の管理が増える。同一Next.jsアプリケーション内で責務を分ける方針を採用する。
