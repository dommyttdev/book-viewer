# ADR-0005: 閲覧用画像をWebPへ変換する

## Status

Accepted

## Context

本システムでは、zip / rar / 7zip形式の自炊本アーカイブからページ画像を取り出し、ブラウザで閲覧する。

原本画像をそのまま配信すると、ファイルサイズが大きくなりやすく、スマートフォン閲覧時の表示速度や通信量に影響する。

一方で、自炊本では漫画や小説の本文、セリフ、細い線の可読性を保つ必要がある。初期想定ではスマートフォン閲覧を主対象とし、拡大表示は想定しない。

## Decision

閲覧用画像はWebPへ変換する。

WebP品質値の既定値は80とし、application.propertiesで変更可能にする。サムネイルも生成し、一覧、検索結果、詳細画面ではサムネイルを利用する。

変換済みWebPとサムネイルはファイル保存領域に保存する。PostgreSQLには生成物の管理情報とページ情報を記録する。

## Consequences

- 原本画像を直接配信する場合に比べ、表示速度と保存容量のバランスを取りやすい。
- WebPは主要ブラウザで広く利用でき、スマートフォン閲覧とも相性がよい。
- 品質値80を既定にすることで、本文やセリフの可読性を確保しつつ、過度な高品質設定を避けられる。
- 変換処理が必要になるため、専用キューと変換ワーカーによる非同期処理が前提になる。
- 画質、最大幅、最大高さ、透過画像、縦長画像、再変換仕様は後続の画像変換設計で詳細化する必要がある。
- 変換済みWebPは派生ファイルとして扱い、必要に応じて原本から再生成できるようにする。

## Alternatives

- 原本画像をそのまま配信する。
  - 変換処理は不要になるが、表示速度、通信量、端末負荷、保存容量の面で不利になりやすい。
- JPEGへ変換する。
  - 対応範囲は広いが、同程度の視覚品質でWebPよりファイルサイズが大きくなりやすい。
- PNGへ変換する。
  - 可逆圧縮だが、写真やグレースケールのページではサイズが大きくなりやすく、閲覧用の既定形式にはしない。
- AVIFへ変換する。
  - 圧縮効率は高いが、変換負荷や運用上の安定性を考慮し、初期方針ではWebPを採用する。
