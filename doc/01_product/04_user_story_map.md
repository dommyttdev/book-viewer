# ユーザーストーリーマップ

## 目的

このドキュメントは、自炊本閲覧Webアプリケーションの利用者行動を軸に、必要な機能を整理するためのユーザーストーリーマップである。

詳細なユーザーストーリー、受入条件、画面仕様、API仕様は、バックログおよび設計ドキュメントで継続的に具体化する。

## 前提

- 一般ユーザは、登録済みの書籍を検索、閲覧、お気に入り管理する。
- 一般ユーザは書籍を保持せず、書籍アップロードも行わない。
- 管理ユーザは、書籍アップロード、メタ情報管理、変換状態確認、ユーザ管理、ロール管理を行う。
- PostgreSQLを正本とし、Elasticsearchは検索用の派生データとして扱う。
- アップロードされた原本ファイルは保存し続ける。
- 画像変換は非同期ジョブとして実行し、変換ワーカーが担当する。
- MVPでは、登録した自炊本を探して読める最小限の流れを優先する。

## 利用者

| 利用者 | 主な目的 |
| --- | --- |
| 一般ユーザ | 登録済みの自炊本を探し、読み、読みかけやお気に入りから再閲覧する。 |
| 管理ユーザ | 自炊本を登録、変換、整理し、一般ユーザが閲覧できる状態を保つ。 |

## MVPの基本利用フロー

1. 一般ユーザが会員登録し、メール認証を完了する。
2. 管理ユーザが管理画面へログインする。
3. 管理ユーザが自炊本アーカイブをアップロードする。
4. バックエンドAPIが原本ファイルを保存し、変換ジョブを作成する。
5. 変換ワーカーがアーカイブを展開し、WebP画像とサムネイルを生成する。
6. 管理ユーザがタイトル、著者、タグ、シリーズなどのメタ情報を整理する。
7. Elasticsearchへ検索用インデックスを反映する。
8. 一般ユーザが一覧または検索から本を探す。
9. 一般ユーザがビューアで本を読む。
10. 一般ユーザの閲覧履歴とお気に入りが保存され、再閲覧に利用できる。

## MVPで優先するストーリー

| 優先度 | ストーリー |
| --- | --- |
| 高 | 一般ユーザとして、会員登録、メール認証、ログイン、ログアウトをしたい。 |
| 高 | 管理ユーザとして、自炊本アーカイブをアップロードしたい。 |
| 高 | 管理ユーザとして、アップロード後の変換状態を確認したい。 |
| 高 | 管理ユーザとして、タイトル、著者、タグ、シリーズを編集したい。 |
| 高 | 一般ユーザとして、本一覧または検索から本を見つけたい。 |
| 高 | 一般ユーザとして、ビューアで本を読みたい。 |
| 高 | 一般ユーザとして、読みかけ位置を保存して再開したい。 |
| 中 | 一般ユーザとして、本をお気に入り登録、解除したい。 |
| 中 | 管理ユーザとして、管理ユーザとロールを管理したい。 |
| 中 | 管理ユーザまたは運用者として、Elasticsearchを再インデックスしたい。 |

## MVP外として扱うストーリー

- 一般ユーザとして、自分の書籍をアップロードしたい。
- 一般ユーザとして、他の一般ユーザと書籍を共有したい。
- 管理ユーザとして、一般ユーザが所有する書籍を閲覧または削除したい。
- 一般ユーザとして、オフラインで本を読みたい。
- 一般ユーザとして、OCRされた本文を全文検索したい。
- 管理ユーザとして、外部書誌データベースからメタ情報を自動取得したい。
- 運用者として、バックアップを取得、復元したい。

## 更新方針

- MVP、Beta、v1.0、将来対応の区分はロードマップ作成時に見直す。
- ユーザーストーリーを追加、変更、削除した場合は、関連するエピック、受入条件、設計ドキュメントも確認する。
- 仕様判断が長期的な影響を持つ場合は、ADRまたは設計ドキュメントに判断理由を残す。
