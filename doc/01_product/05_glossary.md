# 用語集

## 目的

このドキュメントは、自炊本閲覧Webアプリケーションで使用する主要な用語の意味をそろえるための用語集である。

用語の定義は、プロダクト、バックログ、設計、ADR、運用ドキュメントで共通して参照する。仕様変更や新しい概念の追加が発生した場合は、この用語集も更新する。

## 用語

| 用語 | 定義 |
| --- | --- |
| 自炊本 | 紙の本などを利用者自身がスキャンし、画像またはアーカイブファイルとして電子化した本。本アプリでは、管理ユーザがアップロードし、変換処理を経て一般ユーザが閲覧できる対象を指す。 |
| 書籍 | アプリケーション上で管理する本の単位。タイトル、著者、タグ、シリーズ、ページ、原本ファイル、変換済み画像、サムネイルなどを関連づけて扱う。 |
| 原本ファイル | 管理ユーザがアップロードした元のアーカイブファイル。zip / rar / 7zip 形式を主な対象とし、通常運用では保存し続ける。 |
| アーカイブファイル | 複数のページ画像などを1つにまとめたファイル。MVPでは zip / rar / 7zip を受け付ける想定とする。 |
| 展開 | アーカイブファイルからページ画像などの内容物を取り出す処理。rar / 7zip の展開には、変換ワーカーから 7-Zip for Linux コンソール版を外部プロセスとして呼び出す。 |
| 変換済み画像 | 原本ファイルから取り出したページ画像を、Web閲覧向けに変換した画像。既定ではWebP形式、品質値80で生成する。 |
| WebP | Web向けの画像形式。本アプリでは、表示速度と保存容量のバランスを取るため、変換済み画像の既定形式として使用する。 |
| サムネイル | 一覧や検索結果などで書籍を識別しやすくするための小さい代表画像。変換処理の一部として生成する。 |
| ページ | 書籍を構成する1枚の閲覧単位。変換済み画像と表示順を持ち、ビューアで前後に移動して閲覧する。 |
| タイトル | 書籍名。検索、一覧表示、詳細表示で使用する主要なメタ情報。 |
| 著者 | 書籍の作者、作画者、編集者など、書籍に関連する人物または団体。書籍と多対多で関連づける可能性がある。 |
| シリーズ | 複数の書籍をまとめる単位。巻数や並び順、シリーズ概要を持たせることを想定する。 |
| 種別 | 書籍の形態を分類するためのメタ情報。単行本、雑誌、アンソロジーなど、本そのものの種類を管理するために使用する。 |
| タグ | 書籍を分類、整理、検索するための任意のラベル。ジャンル、状態、テーマなどに利用する。 |
| お気に入り | 一般ユーザが再閲覧しやすいように書籍を登録しておく機能、またはその登録状態。同じ本の重複登録は許可しない方針とする。 |
| 閲覧履歴 | 一般ユーザが最後に読んだ書籍、ページ、日時などを記録する情報。読みかけ位置の復元に利用する。 |
| 一般ユーザ | 登録済みの自炊本を検索、閲覧、お気に入り管理する利用者。書籍を保持せず、書籍アップロードも行わない。 |
| 管理ユーザ | 自炊本のアップロード、メタ情報管理、変換状態確認、ユーザ管理、ロール管理などの管理操作を行う利用者。 |
| ロール | 管理ユーザなどに割り当てる権限のまとまり。操作可否を制御するために使用する。 |
| 権限 | 特定の操作を実行できる能力。書籍アップロード、書籍削除、ロール設定、ユーザ停止などの単位で設計する。 |
| 変換ジョブ | アップロードされた原本ファイルを展開し、ページ画像をWebPへ変換し、サムネイルを生成する非同期処理の単位。 |
| RabbitMQ | 変換ジョブをAPI処理から分離して非同期に配送するための専用キュー製品。DBをジョブ配送キューとして使わない方針とする。 |
| 変換ワーカー | RabbitMQから変換ジョブを取得し、アーカイブ展開、画像変換、サムネイル生成を実行するSpring Bootアプリケーション。 |
| バックエンドAPI | フロントエンドから呼び出されるSpring Bootアプリケーション。認証、書籍情報、検索、閲覧、管理操作などのAPIを提供する。 |
| フロントエンドWebアプリケーション | 利用者がブラウザで操作するNext.jsアプリケーション。一般ユーザ向け画面と管理ユーザ向け画面を提供する。 |
| PostgreSQL | メタ情報、ユーザ、権限、ジョブ状態などを保存する正本のデータベース。 |
| Elasticsearch | タイトル、著者、タグ、シリーズなどの検索に使用する検索エンジン。PostgreSQLから再構築可能な派生データとして扱う。 |
| analysis-kuromoji | Elasticsearchで日本語の形態素解析を行うためのプラグイン。本アプリでは日本語検索の基本方針として使用する。 |
| analysis-icu | ElasticsearchでICU normalizerなどを利用するためのプラグイン。本アプリでは全角 / 半角、大小文字、互換文字などの表記揺れ正規化に使用する。 |
| 再インデックス | PostgreSQLの正本データをもとに、Elasticsearchの検索インデックスを作成し直す処理。全件または書籍単位で実行できるようにする。 |
| モジュラーモノリス | 単一アプリケーション内で明確なモジュール境界を保つ構成方針。初期の開発と運用を単純にしつつ、将来的なAPI / Worker分離を可能にする。 |
| MVP | Minimum Viable Productの略。本アプリでは、自炊本を登録し、変換し、探し、読むための最小限の流れが成立する初期範囲を指す。 |

## 更新方針

- 新しい用語を設計書やADRで使う場合は、必要に応じてこの用語集へ追加する。
- 既存用語の意味が変わる場合は、関連する設計書、ADR、ユーザーストーリーもあわせて確認する。
- 略語や外部技術名は、初出時に役割と本アプリでの扱いを明記する。
